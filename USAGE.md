# Flash Memo 사용법

**Flash Memo**는 브라우저에서 바로 쓰는 가벼운 메모 앱입니다.  
생각이 떠오르면 적고, 태그·검색으로 다시 찾을 수 있습니다.  
데이터는 **이 브라우저(이 기기)에만** 저장됩니다.

## 1. 시작하기

### 일반 사용자 (배포된 주소가 있을 때)

1. Chrome 또는 Edge에서 **배포 URL**을 엽니다. (예: `https://xxxx.vercel.app`)
2. 화면 상단 **⚡ Flash Memo**가 보이면 준비 완료입니다.
3. 입력 칸에 글을 쓰고 **✓ 저장**을 누릅니다.

### 처음 열면

- 입력 칸에 커서가 자동으로 들어갑니다.
- 메모가 없으면 목록에 **「아직 메모가 없어요」**가 표시됩니다.

---

## 2. 메모 쓰기·저장

| 단계 | 할 일 |
|------|--------|
| 1 | 가운데 입력 칸에 내용을 적습니다. |
| 2 | **✓ 저장** 버튼을 누르거나 **Ctrl+Enter** (Mac: **⌘+Enter**) |
| 3 | 저장된 메모는 아래 **메모 목록**에 나타납니다. |

**참고**

- 빈 칸만 있으면 저장되지 않습니다.
- 저장 후 입력 칸은 비워지고, 다시 바로 쓸 수 있습니다.

## 3. 꾸미기 (간단 마크다운)

입력 칸 아래 안내와 같이, 아래 문법만 지원합니다.

| 입력 예시 | 화면에서 |
|-----------|----------|
| `**중요**` | **굵게** |
| `[구글](https://google.com)` | 클릭 가능한 링크 (http/https만) |
| `- [ ] 할 일` | ☐ 체크 안 된 할 일 |
| `- [x] 완료` | ☑ 완료된 할 일 |

여러 줄은 **줄바꿈** 그대로 반영됩니다.

## 4. 태그와 즐겨찾기

### 태그 (#)

- 본문에 `#업무`, `#아이디어`처럼 **띄어쓰기 없이** 적습니다.
- 저장된 메모에서 **태그 글자**를 클릭하면, 그 태그만 모아 봅니다.
- 상단에 **「태그 #이름」**과 **필터 해제**가 나타납니다.

**# 버튼**

- 입력 칸 옆 **#** 버튼을 누르면 커서 위치에 `#`가 들어갑니다.

### 즐겨찾기 태그

| 동작 | 방법 |
|------|------|
| 자주 쓰는 태그 등록 | **즐겨찾기** 옆 **+** → 태그 이름 입력 → Enter |
| 본문에 넣기 | 즐겨찾기 칩 `#이름` 클릭 |
| 삭제 | 칩 옆 **×** |

- 즐겨찾기는 **최대 12개**까지 등록할 수 있습니다.
- 편집 모드에서도 같은 즐겨찾기 줄이 보입니다.

## 5. 목록 보기·찾기

메모가 하나라도 있으면 상단·목록에 아래 기능이 나타납니다.

### 검색

- 상단 **검색** 칸에 단어를 입력하면, **본문·태그**에 포함된 메모만 보입니다.

### 정렬·개수

| 설정 | 설명 |
|------|------|
| **최신순 / 오래된순** | 시간 기준 정렬 (고정 메모는 항상 위) |
| **표시 N개** | 한 페이지에 보이는 메모 수 (1~100) |

### 페이지

- 메모가 많으면 하단 **이전 / 다음**으로 넘깁니다.
- `현재 페이지 / 전체 페이지`와 범위가 함께 표시됩니다.

---

## 6. 메모 관리 (고정·수정·삭제)

각 메모 카드 오른쪽 아래 버튼:

| 버튼 | 기능 |
|------|------|
| ☆ / ⭐ | **상단 고정** (⭐이면 고정됨, 목록 맨 위 유지) |
| ✏️ | **편집** — 내용 수정 후 **저장** 또는 **취소** |
| 🗑 | **삭제** — 한 번 누르면 「다시 누르면 삭제」 → **3초 안에** 다시 누르면 삭제 |

**편집 중**

- **Ctrl+Enter** (Mac: **⌘+Enter**): 저장  
- **Esc**: 취소

## 8. 자주 묻는 질문

### 메모는 어디에 저장되나요?

- 이 브라우저의 **로컬 저장소(localStorage)**에만 저장됩니다.
- **같은 주소·같은 브라우저**에서 다시 열면 그대로 남습니다.
- 다른 PC·다른 브라우저·시크릿 모드에서는 **보이지 않을 수 있습니다**.

### 백업이나 계정 로그인이 있나요?

- 없습니다. 중요한 내용은 별도로 복사·백업해 두는 것을 권장합니다.

### 삭제한 메모를 되돌릴 수 있나요?

- **되돌리기(실행 취소)는 없습니다.** 삭제 전에 한 번 더 확인하세요.

### 검색과 태그 필터는 같이 쓰나요?

- 검색 칸을 바꾸면 태그 필터가 해제됩니다.
- 태그를 클릭하면 검색 칸에 태그 이름이 들어가며, 해당 태그만 필터됩니다.

### 친구에게 배포 URL을 내도 되나요?

- **가능합니다.** 링크는 **앱을 소개**하는 것이지, **내 메모를 공유**하는 것이 아닙니다.
- 메모는 각자 브라우저에만 저장됩니다. 자세한 설명은 **[SECURITY.md](./SECURITY.md)** 를 참고하세요.

## 9. PC에 앱처럼 설치 (PWA)

**HTTPS**로 열린 주소(배포 URL)에서 설치하는 것이 가장 안정적입니다.

### Chrome / Edge (Windows)

1. 배포 URL을 엽니다.
2. 주소창 오른쪽 **앱 설치** 또는 메뉴 **앱 → 이 사이트를 앱으로 설치**를 선택합니다.
3. 바탕화면·시작 메뉴에 **Flash Memo** 아이콘이 생깁니다.

**알아두기**

- `localhost`에 설치한 것과 **배포 URL**에 설치한 것은 **별개**입니다. 매일 쓸 때는 배포 URL로 설치하는 것을 권장합니다.
- 개발 서버(`npm run dev`)에서는 설치·오프라인 동작이 제한될 수 있습니다.

## 10. 개발자: 프로젝트 실행

프로젝트 폴더: `myMemo_2606` (여기에 `package.json`이 있습니다.)

### 최초 1회

```powershell
cd d:\Projects\mymemo_1\myMemo_2606
npm install
```

### 용도별 명령

| 목적 | 명령 | 주소 |
|------|------|------|
| UI 개발 | `npm run dev` | http://localhost:5173 |
| 배포 전 확인 | `npm run build` 후 `npm run preview` | http://localhost:4173 |
| 매일 사용 (권장) | Vercel 등에 배포한 **HTTPS URL** | preview는 터미널을 끄면 종료 |

### Vercel 배포 (요약)

1. GitHub에 저장소 push  
2. [vercel.com](https://vercel.com) → GitHub 연동 → 이 repo Import  
3. Framework: **Vite**, Build: `npm run build`, Output: **`dist`**  
4. 배포 URL로 접속 후 [9. PC에 앱처럼 설치](#9-pc에-앱처럼-설치-pwa) 진행  

코드를 수정해 push하면, 연동 시 Vercel이 자동으로 다시 빌드·배포합니다.

# Flash Memo 보안 안내

이 문서는 **배포 URL(`https://my-memo-2606.vercel.app` 등)을 친구에게 공유해도 되는지**를 중심으로,  
프로젝트 코드를 기준으로 정리한 보안·프라이버시 설명입니다.

**작성 배경:** 공개 메모 서비스에 흔한 보안 조언(“링크를 받으면 내용이 다 보인다” 등)이  
이 앱에는 그대로 적용되지 않는다는 점을 명확히 하기 위해 정리했습니다.

## 2. 우리가 나눈 대화 요약

### 처음 질문 (일반적인 보안 조언)

다른 AI·자료에서 흔히 나오는 내용:

- 공개 메모 서비스면 링크를 받은 사람이 **전체 내용**을 볼 수 있음
- 로그인 없이 수정·삭제가 가능하면 **악의적 변경** 위험
- URL에 **비밀 토큰·관리자 키**가 있으면 공유하면 안 됨
- `*.vercel.app`은 **HTTPS**라 전송 구간은 암호화됨

→ **공개 게시판형 서비스**에는 맞는 상식입니다.

### 이 프로젝트에 맞춘 재점검

Flash Memo 코드를 보면:

- 메모는 **서버·DB가 아니라 `localStorage`에만** 저장
- **API 호출·로그인·DB 없음**
- URL로 메모를 넘기는 기능(`?memo=...`) 없음
- Analytics·Sentry 등 **외부로 메모를 보내는 코드 없음**

### 합의된 결론

- 친구에게 URL을 내도 **당신 메모는 전송되지 않음**
- 친구는 **같은 웹앱**만 열고, **자기 브라우저**에 메모가 쌓임
- **계산기 사이트 링크**를 보내는 것과 비슷 — 상대에게 **내 사용 기록이 넘어가지 않음**
- 다만 메모는 **브라우저에 평문** → 기기·브라우저를 다른 사람이 쓰면 노출 가능

## 3. 이 앱의 실제 구조 (코드 기준)

| 항목 | Flash Memo |
|------|------------|
| 저장 위치 | 브라우저 `localStorage` (`flashMemos`, `flashMemoFavoriteTags` 등) |
| 서버·DB | **없음** |
| 로그인·계정 | **없음** |
| 네트워크 (메모 전송) | **없음** (`fetch` 등 미사용) |
| 배포 (Vercel) | 빌드된 **정적 파일**(HTML/JS/CSS)만 제공 |
| PWA | `vite-plugin-pwa` — 오프라인·설치용, 메모는 여전히 로컬 |

**데이터 흐름**

```
[사용자 입력] → [브라우저 메모리] → [localStorage] → (끝)
                    ↑
              Vercel은 앱 코드만 전달, 메모는 거치지 않음
```

## 4. URL 공유: 안전한 이유

`https://my-memo-2606.vercel.app` 링크가 하는 일:

1. 친구 PC의 브라우저가 **앱 JavaScript**를 받아 실행
2. 친구 브라우저의 **빈(또는 기존) localStorage**에서 메모를 읽고 씀
3. **당신 PC의 localStorage에는 접근 불가** (브라우저 보안 정책)

따라서:

| 행위 | 의미 |
|------|------|
| URL 공유 | **앱 추천** (“이 메모 앱 써 봐”) |
| URL 공유 ≠ | **메모 공유** (“내 메모 봐”) |

친구가 당신 메모를 **읽거나 삭제할 수 없습니다** (원격으로).

## 5. 일반 조언과의 차이

| 일반 조언 (공개 메모 서비스) | Flash Memo |
|------------------------------|------------|
| 링크 받으면 **내 글**이 보임 | 링크는 **앱만** 열림, **내 글은 안 보임** |
| 누구나 **남의 글** 수정·삭제 | 각자 **자기 브라우저** 안에서만 수정·삭제 |
| API Key·DB 쓰기 권한 점검 | **해당 없음** (백엔드 없음) |

나중에 **서버 저장·공유 링크·로그인**을 붙이면, 위 일반 조언이 **다시 중요**해집니다.

## 6. 지금 주의해야 할 점

### 사용자로서

1. **민감 정보**  
   비밀번호, 계좌, 개인정보는 “유출될 수 있다”고 가정하고 넣지 않기.

2. **같은 PC·브라우저 공유**  
   메모는 평문 `localStorage`. 가족·동료가 같은 Windows 계정·같은 Chrome 프로필을 쓰면 DevTools 등으로 열람 가능.

3. **백업 없음**  
   브라우저 데이터 삭제, 시크릿 모드, 다른 기기 → 메모가 **없거나 비어 있음**. 중요한 내용은 별도 복사·백업.

4. **URL 공유의 의미**  
   친구에게 보낼 때 “내 메모 보여주는 링크”가 아니라 **“앱 설치/사용 링크”**임을 알기.

5. **PWA 설치**  
   `localhost`에 설치한 것과 **배포 URL**에 설치한 것은 별개. 데이터도 브라우저·출처(origin)별로 분리됨.

### 운영자(본인)로서

1. **Vercel·GitHub 계정**  
   배포 권한이 있으면 악성 코드를 올릴 수 있음. 2단계 인증·강한 비밀번호 권장.

2. **의존성·빌드**  
   `npm` 패키지는 주기적으로 업데이트. 알 수 없는 스크립트를 코드에 넣지 않기.

3. **`.env`·API Key**  
   현재는 없음. 나중에 백엔드를 붙일 때 **프론트에 Key 노출 금지** 규칙 지키기.

## 7. 앞으로 알아두고 지켜야 할 점

### 구조가 바뀌면 위험 모델도 바뀜

아래를 추가하는 순간, “링크만 내도 안전” 전제는 **깨질 수 있습니다.**

| 추가 기능 | 새로 생기는 질문 |
|-----------|------------------|
| 서버·DB에 메모 저장 | 누가 읽고 쓸 수 있나? DB 규칙이 열려 있나? |
| 공유 링크 (`/share/abc`) | 링크만 알면 내용이 보이나? |
| URL 쿼리 (`?memo=`) | 주소창·기록·Referer에 내용이 남나? |
| 로그인 없는 쓰기 API | 남이 내 메모를 지울 수 있나? |
| Google Analytics, Sentry | 메모 **본문**이 전송되나? |
| 클라우드 동기화 | 계정 탈취·서버 유출 시 영향 범위는? |

### 지금은 해당 없는 예외 (코드 확인됨)

- URL에 메모 내용 넣기
- 서버 저장
- Analytics / Sentry로 메모 전송
- (앱 밖) 악성 확장·악성코드가 PC를 감시하는 경우 → **어떤 웹앱이든** 해당

### HTTPS (`vercel.app`)

- 브라우저 ↔ Vercel 사이 **전송 구간**은 암호화됨
- **저장된 메모**를 암호화해 주지는 **않음** (로컬 평문)

## 9. 자주 묻는 질문

### 친구에게 `my-memo-2606.vercel.app` 링크를 내도 되나?

**된다.** 지금 구조에서는 **앱 소개**이며, **당신 메모는 넘어가지 않는다.**

### 친구가 내 메모를 볼 수 있나?

**링크만으로는 불가.** 각자 자기 브라우저 `localStorage`만 사용.

### HTTPS면 메모도 암호화되나?

**아니다.** HTTPS는 **전송**만 보호. 저장은 브라우저 **평문**.

### 회사·학교 PC에서 써도 되나?

가능하지만, **기기 정책·공유 PC**면 메모 노출·데이터 삭제 정책을 먼저 확인.

### 나중에 “친구와 메모 공유”를 만들고 싶다면?

의도적으로 **서버·권한·공유 링크** 설계가 필요하고, 그때는 [§8 체크리스트](#8-기능을-추가할-때-체크리스트)를 반드시 거치기.
